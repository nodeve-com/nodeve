# grimoire cadence field — `update_interval_ms`

The grimoire `defaults` cadence field is `update_interval_ms` — integer milliseconds, named for the _rate_ a fresh data point is expected/emitted, not the fetch strategy. It was renamed from an earlier fetch-named, seconds-valued field.

**Why:** Two grounded corrections from the user:

- **ms, not seconds.** In industrial Modbus-master tooling the convention is integer milliseconds — Kepware/KEPServerEX (default 1000 ms), Ignition (250 ms), modpoll/mbpoll (default 1000 ms), Modbus Poll. Seconds only appears one tier up in the IoT/data-collection layer (HA `scan_interval`, Telegraf durations). grimoire (with its consuming gateway) _is_ a Modbus polling master, so it follows the master-layer convention. ("Seconds reads more naturally" was an aesthetic borrowed from the wrong tier.)
- **fetch-neutral name, not "poll".** "Poll" names a fetch _strategy_; grimoire is protocol-neutral (the protocol union grows past Modbus to VE.Direct, canboat/NMEA) and a discovered NMEA-2000 network isn't polled — frames arrive on the bus. Baking "poll" into the field name is the same leak as naming a consumer (the [consumer-agnostic guard](../../../scripts/guard-grimoire-agnostic.sh) enforces that). `update_interval_ms` reads correctly for both a polling driver (poll rate) and a push driver (throttle/expected-update rate), and across both `publish: changed|always` modes. `interval_ms` alone was rejected as too bare next to `publish`.

**How to apply:** Use `update_interval_ms` (integer ms). A pre-commit guard ([`scripts/guard-interval-naming.sh`](../../../scripts/guard-interval-naming.sh), run by lefthook) fails any staged grimoire content that reintroduces a fetch-named or seconds-suffixed interval field, so the old name can't return — mirrors the consumer-agnostic guard. The guard targets the field NAME only: the verb "poll" stays fine in grimoire prose (the inverter genuinely polls the meter bus, `mbpoll`, "NOT polled" comments), so a blanket "poll" guard is NOT viable. The consuming gateway reads the value directly as ms — no `* 1000`. Broader lesson the user reinforced: ground naming/unit choices in the conventions of the layer the code actually lives in, not first-principles aesthetics.
